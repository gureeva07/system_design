## Роли пользователей
**Неавторизованный пользователь (гость)** — незарегистрированный или не вошедший в систему посетитель

**Авторизованный пользователь (отправитель)** — зарегистрированный участник сети. Имеет доступ к полному функционалу (инициирующий отправку сообщений и публикацию записей)

**Авторизованный пользователь (получатель)** — зарегистрированный участник сети, получающий сообщения и уведомления

**Сотрудник технической поддержки** — внутренний пользователь системы. Имеет права модератора + сотрудника тех.поддержки

## Внешние системы для интеграции
**Email-сервис** — используется для отправки транзакционных писем: подтверждение регистрации, уведомления об авторизации

**Push-сервис** — отвечает за доставку push-уведомлений на мобильные и веб-устройства авторизованных пользователей: уведомления о новых публикациях на стене и входящих личных сообщениях

**Система рекламы** — интегрируется с сервисом рекомендаций

## Use cases:
- **UC-1: Регистрация аккаунта**  
  Актор: Гость  
  Что: Регистрация нового пользователя с заполнением профиля

- **UC-2: Авторизация**  
  Актор: Гость -> Авторизованный пользователь  
  Что: Вход в систему через логин и пароль для получения доступа к функционалу

- **UC-3: Просмотр публичного профиля и стены**  
  Актор: Гость  
  Что: Просмотр публичных страниц пользователей и их записей без авторизации

- **UC-4: Поиск пользователя по логину**  
  Актор: Авторизованный пользователь  
  Что: Поиск через UI по точному логину пользователя

- **UC-5: Поиск пользователя по маске имени и фамилии**  
  Актор: Авторизованный пользователь  
  Что: Поиск через UI по частичному совпадению имени и/или фамилии

- **UC-6: Добавление записи на стену**  
  Актор: Авторизованный пользователь  
  Что: Публикация текстовой записи на своей стене

- **UC-7: Загрузка стены пользователя**  
  Актор: Авторизованный пользователь  
  Что: Просмотр ленты записей своей стены или стены другого пользователя

- **UC-8: Отправка P2P-сообщения**  
  Актор: Авторизованный пользователь -> Авторизованный пользователь  
  Что: Отправка личного сообщения другому пользователю через чат

- **UC-9: Получение списка сообщений**  
  Актор: Авторизованный пользователь  
  Что: Просмотр истории переписки с конкретным пользователем

- **UC-10: Уведомления о новых постах и сообщениях**  
  Актор: Social Network -> Push Service -> Авторизованный пользователь (получатель)  
  Что: Отправка push-уведомлений при появлении новых записей на стене или входящих сообщений

- **UC-11: Уведомление об авторизации**  
  Актор: Social Network -> Email Service -> Гость  
  Что: Отправка email-уведомления при регистрации или входе в аккаунт

- **UC-12: Показ рекламы**  
  Актор: Social Network -> Recommendation Service -> Реклама -> Авторизованный пользователь  
  Что: Recommendation Service анализирует активность пользователя и передаёт профиль интересов во внешнюю рекламную систему для подбора релевантных материалов

## Динамическая диаграмма — Отправка P2P-сообщения
1. Авторизованный пользователь (отправитель) вводит текст и нажимает «Отправить» в веб-приложении
2. `Web App` отправляет `POST /api/chats/messages` на `API Gateway`
3. `API Gateway` перенаправляет запрос в `Chat Service`
4. `Chat Service` обращается к `User Service` для проверки существования получателя по ID
5. `User Service` выполняет поиск пользователя в `User DB`
6. `User DB` возвращает данные пользователя
7. `User Service` сообщает `Chat Service`, что получатель найден
8. `Chat Service` сохраняет сообщение в `Chat DB` со статусом «Отправлено»
9. `Chat DB` подтверждает сохранение и возвращает ID сообщения
10. `Chat Service` отправляет уведомление о новом сообщении через `Push Service`
11. `Push Service` доставляет push-уведомление авторизованному пользователю (получателю)
12. `Chat Service` возвращает успешный ответ с ID сообщения в `API Gateway`
13. `API Gateway` передаёт ответ в `Web App`
14. `Web App` отображает сообщение авторизованному пользователю (отправителю) как доставленное
